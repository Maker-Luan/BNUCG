## 二、核心实现简述

**1. 场景初始化与 GPU 状态同步**
布料网格设为 20×20（共 441 个质点），初始化位置（平铺于 XZ 平面）、速度（零）、受力（零）以及结构弹簧拓扑（水平、垂直连接）。为保证 GPU 多线程数据一致性，将初始化拆分为三个独立 `@ti.kernel`：
- `init_positions()`：设置质点初始坐标；
- `init_springs()`：建立弹簧连接关系（原长、刚度系数）；
- `init_render_indices()`：为三角形网格渲染预计算顶点索引。
在 Python 主循环中按顺序依次调用，避免并行竞态。

**2. 力学计算与防爆处理**
在 `compute_forces()` 中（声明为 `@ti.func`，强制内联）：
- 累加重力 `(0, -9.8, 0)`；
- 遍历弹簧连接，根据胡克定律计算弹力，并使用 `ti.atomic_add` 安全累加至两端质点；
- 施加阻尼力 `-k_d * v`（与速度成正比）。
同时编写 `clamp_velocity()` 函数，对更新后的速度模长进行截断（上限如 20.0），防止显式欧拉等弱稳定性方法引发数值爆炸。

**3. 三种数值积分方法的独立 Kernel 实现**
为追求极致性能，将受力计算、速度更新与位置更新合并至同一个 `@ti.kernel` 中，每帧仅启动一次 Kernel：
- **显式欧拉（Explicit）**：先更新位置（用旧速度），再更新速度（用旧加速度）；
- **半隐式欧拉（Semi-Implicit）**：先更新速度（用旧加速度），再更新位置（用新速度）——该方法具有更好的能量守恒特性；
- **隐式欧拉（Implicit）**：由于加速度依赖于未来位置，采用定点迭代法（固定 3 次迭代）近似求解，提升稳定性。

**4. UI 交互面板**
通过 `ti.ui.Window` 的 `GUI` 组件：
- 提供三个按钮（Button），分别切换至三种积分方法，并高亮当前选中的方法；
- 提供“暂停/继续”按钮，控制模拟进程；
- 提供“重置”按钮，将布料恢复至初始平铺状态，便于重复对比实验。

## 四、实验总结

本次实验基于质点-弹簧系统实现了可交互的布料物理模拟，并在同一框架下对比了三种数值积分方法。核心收获如下：

- **物理建模**：深入掌握了胡克定律与阻尼力的具体实现，以及多质点弹簧网络的并行累加技巧（`atomic_add` 的必要性）；
- **数值方法认知**：通过实际运行直观验证了显式欧拉在较大时间步长下极易爆炸、半隐式欧拉在多数场景下兼顾效率与稳定、隐式欧拉（迭代近似）最为鲁棒但计算开销稍大——这与理论分析完全吻合；
- **GPU 编程优化**：领会到“多 Kernel 拆分初始化”对避免数据竞争的重要性，以及将力学计算内联至主 Kernel（`@ti.func`）对减少函数调用开销的显著收益；
- **工程完整性**：借助 GGUI 按钮与滑块，实现了可灵活切换方法、调节参数、重置场景的完整工具，为后续更复杂的物理仿真（如碰撞处理、弯曲弹簧）奠定了坚实基础。
